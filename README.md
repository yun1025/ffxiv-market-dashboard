# FFXIV Market Dashboard

파이널판타지14 글로벌 서버의 거래 게시판 시세를 검색하고 비교하는 웹 대시보드입니다.

## 왜 만들었나요

파이널판타지14를 플레이하며 서버별 시세를 비교할 도구가 필요하다고 느꼈고,
실무에서 다뤘던 프론트엔드 기술을 활용해 직접 만들어보고 싶어 시작했습니다.

## 주요 기능

- 아이템 이름으로 검색 (디바운스 적용)
- 서버별 실시간 최저가 비교
- 가격 추이 차트
- 거래 히스토리 조회

## 기술 스택

- Next.js 15 (App Router)
- TypeScript
- Tailwind CSS
- Recharts
- XIVAPI v2, Universalis (외부 API, 별도 API 키 불필요)

## 아키텍처
```
types → lib → hooks → components → app
```

API 응답 구조를 그대로 컴포넌트에서 쓰지 않고, `lib/`에서 내부용 타입으로 변환하는 계층을 뒀습니다.
외부 API의 응답 구조가 바뀌거나 예상과 다르게 와도 컴포넌트 코드에 영향이 가지 않도록 설계했습니다.

## 실행 방법

```bash
npm install
npm run dev
```

## 트러블슈팅

- **XIVAPI 응답 구조 오류**: `LevelItem` 필드가 단순 숫자가 아니라 다른 데이터 시트를 참조하는
  관계형 객체(`{ value, sheet, row_id }`)로 와서 React 렌더링 에러가 발생. 실제 API 응답을 직접
  확인한 뒤 타입을 관계형 구조에 맞게 수정하고, `.value`만 추출해 사용하도록 변환 로직을 추가했습니다.
- **여러 서버 동시 조회**: 서버 8개를 순차 조회하면 응답이 느려지는 문제가 있어
  `Promise.allSettled`로 병렬 처리했습니다. 일부 서버 조회가 실패해도 나머지 결과는
  정상적으로 보여주기 위해 `Promise.all` 대신 `allSettled`를 선택했습니다.
- **경쟁 상태(race condition) 방지**: 아이템을 빠르게 여러 번 클릭할 경우 먼저 보낸 요청이
  나중에 도착해 최신 화면을 이전 데이터로 덮어쓰는 문제가 있어, 훅 내부에 `isCancelled` 플래그를
  두어 오래된 요청의 결과는 무시하도록 처리했습니다.
- **동적 라우트 설정**: Next.js 15의 동적 라우트(`params`가 Promise로 전달)에 맞춰
  `[id]` 폴더 구조와 `use()` 훅을 사용해 라우트 파라미터를 처리했습니다.

## 아쉬운 점

- 한국 서버는 별도 퍼블리싱 구조로 운영되어 공식 API가 존재하지 않아, 글로벌 서버만 지원합니다.
- 검색은 XIVAPI가 지원하는 영어/일본어/독일어/프랑스어로만 가능합니다.
- 현재는 북미 Aether 데이터센터를 기준으로 고정되어 있으며, 추후 데이터센터 선택 기능을 추가할 예정입니다.
